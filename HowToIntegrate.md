Призначення підсистеми

Команда B3 (Runtime) відповідає за інтеграцію двох компонентів:

A5 — Lib.Sampling

A10 — Lib.ChatConsole

Підсистема Runtime забезпечує інтерактивну генерацію тексту в чаті через REPL-консоль, де:

модель генерує оцінки наступного токена;

семплер (Sampler) вибирає токен з урахуванням параметрів;

користувач взаємодіє з системою через консоль.

Метою підсистеми B3 є створення універсального runtime-шару, який:

не залежить від конкретної моделі;

дозволяє підключати будь-яку модель через ITextGenerator;

забезпечує контроль генерації через параметри (temperature, topK, seed);

надає зручний REPL-інтерфейс для користувача.

Підсистема B3 реалізує всі вимоги:

інтеграція Sampler + ChatRepl

повний pipeline генерації

підтримка команд REPL

model-agnostic архітектура

документ “How to integrate”

Підсистема B3 Runtime є точкою взаємодії користувача з системою.

Вона не залежить від типу моделі;

використовує єдиний контракт ITextGenerator;

керує генерацією через ReplOptions;

відокремлює UI від логіки генерації.

Що очікується від інших команд на етапі 3

Щоб B3 можна було легко підключити, суміжна команда має надати:

Мінімум

модель або обгортку моделі;

метод отримання оцінок наступного токена;

токенізатор;

метод decode назад у текст.

Бажано

єдиний адаптер `ModelTextGenerator`, який уже реалізує `ITextGenerator`;

приклад створення моделі й токенізатора;

тестовий маленький корпус/чекпоінт для демо;

фіксований seed для відтворюваних перевірок.
